Understand RAG

This simulator visually demonstrates how Retrieval-Augmented Generation (RAG) works — from document ingestion and chunking to embeddings, vector retrieval, and final answer generation using an LLM.

1

Documents

Raw dataset or knowledge source is uploaded.

2

Chunking

Large documents are split into smaller chunks.

3

Embeddings

Chunks are converted into vector embeddings.

4

Vector DB

Embeddings are stored for semantic retrieval.

5

Retrieval

Relevant chunks are retrieved based on query similarity.

6

LLM Response

Retrieved context + query are sent to the LLM.

Knowledge Base

What is Chunking?

Chunking breaks large documents into smaller manageable pieces so retrieval becomes more accurate.

What are Embeddings?

Embeddings convert text into numerical vectors so semantic similarity can be calculated.

Why Retrieval?

Retrieval helps the LLM use relevant external context instead of depending only on training memory.
